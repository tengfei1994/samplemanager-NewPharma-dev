# Development Workflow

## Normal Cycle

1. Pull selected remote files when context is needed.
2. Make local changes in `src`, `samplemanager`, or `docs`.
3. Build the local package.
4. Deploy to the remote VGSM instance.
5. Fetch logs and update validation notes.
6. Commit the change to GitHub.

## Source of Truth

The GitHub repository is the source of truth. Remote server changes should be
pulled back deliberately and reviewed before being committed.
